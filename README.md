# Проект "Homework 10_1"
## Описание:
Проект "Homework 10_1" - это домашнее задание для работы с данными о банковских операциях пользователя.
В этом проекте реализованны функции для маскировки номера и счета, а так же сортировка списка словарей 
по дате или ключу.
## Установка:
1.Клонируйте репозиторий:
```
git clone https://github.com/UncleSerezha/PythonProjectDZ
```
2. Установите зависимости:
```
pip install -r requirements.txt
```
### Использование:
1. В модуле "masks.py" в функцию "get_mask_card_number" необходимо ввести номер карты. 
Получится маска номера по правилу XXXX XX** **** XXXX, где X - это цифра номера.

2. В модуле "masks.py" в функцию "get_mask_account" необходимо ввести номер карты.
Получится маска номера по правилу **XXXX, где X - это цифра номера.

3. В модуле "widget.py" в функцию "mask_account_card" необходимо ввести номер карты или счет.
В зависимости от того, что вы ввели- счет или номер, будет выбрана маска по правилу 
XXXX XX** **** XXXX или **XXXX, где X - это цифра номера.

4. В модуле "generatoes.py" в функцию "filter_by_currency" вводится список словарей, представляющих 
транзакции. Функция должна возвращать итератор,который поочередно выдает транзакции, 
где валюта операции соответствует заданной (например, USD). 
Пример ввода filter_by_currency(list_dictionary, 'USD').

5. В модуле "generatoes.py" в функцию "transaction_descriptions" вводится список словарей с 
транзакциями и возвращает описание каждой операции по очереди.

6. В модуле "generatoes.py" в функцию "card_number_generator" вводится диапазон, который выдает номера банковских карт
в формате XXXX XXXX XXXX XXXX, где X — цифра номера карты. Пример ввода: card_number_generator(1, 3). 
Пример вывода: 0000 0000 0000 0001, 0000 0000 0000 0002, 0000 0000 0000 0003.

7. В модуле "decorators.py" функция "log" декоратор, который автоматически логирует начало и конец выполнения функции,
а также ее результаты или возникшие ошибки.

8. В модуле "utils.py" функция "json_file" принимает на вход путь до JSON-файла и возвращает список словарей с данными 
о финансовых транзакциях. Если файл пустой, содержит не список или не найден, 
функция возвращает пустой список. В этом же модуле есть функция "transaction_amount". Она принимает на вход транзакцию
и возвращает сумму транзакции (amount) в рублях, тип данных — float. Если транзакция была в USD или EUR, 
происходит обращение к внешнему API для получения текущего курса валют и конвертации суммы операции в рубли.

9. В пакете "tests" есть модули с проверочными тестами для каждой функции из пакета "src".Так же в пакете
"tests" есть модуль "conftest" c колекцией фикстур.Отчет о покрытии в HTML-формате будет сгенерирован в 
папке "htmlcov" и храниться в файле с названием "index.html".
